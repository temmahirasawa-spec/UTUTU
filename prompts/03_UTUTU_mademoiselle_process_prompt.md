# UTUTU 実績詳細：マドモアゼルクロケット（プロセス・ドキュメンタリー型）実装プロンプト

このプロンプトは、既存の UTUTU 実績詳細ページ（`/projects/[slug]`）の仕組みの上に、**マドモアゼルクロケット（Mademoiselle Croquette）専用の、特別に長い"制作プロセス紹介型"の詳細ページ**を作るためのものです。

他ブランドの詳細ページは「ブランドストーリー型」ですが、この案件は **AIとの壁打ちで作り上げた制作プロセスそのものが見どころ**なので、**全プロセスを削らず縦長で見せる"ドキュメンタリー型"**にします。

## 大前提

- **既存の実績詳細ページのトーン&マナー・コンポーネント・スクロール演出の文法を踏襲する。** 新しい書体・色・モーション語彙は足さない。UTUTU 既存（ダーク基調 / vermilion `#E8482A` 一点 / Helvetica系＋Noto Sans JP＋JetBrains Mono / 引き算）に従う。
- **原稿は日本語主体**（英主・日従ルールが残るヒーロー等は英字タイトル＋日本語サブ）。日本語は明朝不使用 / `word-break: keep-all` / `line-break: strict`。
- **このページだけは長くてよい。** 他ページより冗長・縦長OK。全プロセス（後述の12段階）を1つも削らずに見せる。
- **画像はすべてダミー（プレースホルダ）で組む。** 後でユーザーが差し込む。下記「画像の扱い」参照。
- データ構造は既存の `lib/projects.ts`（`Project` / `ProjectBlock` 型）を使う。**このページの分量に対応するため、必要なら `ProjectBlock` の type を増やしてよい**（例：`processStep` ステップ番号付きブロック）。ただし既存の他ブランドページが壊れないよう後方互換で。

### C. 使用した手法・テクノロジー（このページの個性）
"AI時代の作り方"を見せるセクション。使ったツールを、役割とともに整理して見せる。既存のサービス/タグ表現（mono ラベル＋短い説明）の作法で。
- **ChatGPT**：コンセプト言語化・ストーリー・タグライン・プロンプト設計・資料構成の壁打ち。特に「曖昧なイメージを制作可能な要件に分解」する役割。（例の対が効く：「フランスっぽいけど古臭くない」「手描きだけど素人っぽくない」「キャラだけどキャラ商売っぽくしない」「ロゴとして使える抽象度」）
- **Midjourney / Stable Diffusion / ComfyUI 系**：ビジュアル探索・キャラクター案・ロゴ方向性・看板娘の雰囲気づくり。イラストを参照画像に構図/服装/世界観を保ったまま生成。
- **Photoshop / Illustrator / Figma**：AI生成を最終成果物にせず、ロゴ運用・ロックアップ・色・余白・視認性を整え、質感・合成・キービジュアル化。
- このセクションの締めの一言（日本語）：「AIで作る、ではなく、**AIを混ぜたアートディレクション**。AIは探索エンジン、最終判断と仕上げは人。」

### D. プロセス本編（全工程・削らない・縦長OK）
**ここが主役。** 以下の段階を**1つも省略せず順番に**、それぞれ「ステップ番号＋見出し＋本文＋画像（ダミー）」のブロックで見せる。各ステップに画像が複数入る前提で、画像が増えても破綻しないレイアウトにする。

工程（番号と見出しは下記を使用。本文は提供素材をUTUTUトーンの日本語に整える）：

1. **業態の整理** — クリームコロッケ専門店として何を売るブランドか定義。カッパ横丁という出店イメージ。「ただの揚げ物屋にしない」前提。
2. **名前とタグラインの確定** — Mademoiselle Croquette ／ Crisp Out, Cream In。洋食・フランス感・商品特徴（外サクサク・中クリーム）を圧縮。
3. **ブランドストーリーの構築** — フランスで腕を磨いた女性シェフが、日本の洋食文化・クリームコロッケに惹かれ日本で店を開く、という物語。これにより「商品」から「登場人物のいる店」へ。
4. **ロゴ・キャラクターの探索** — 少女の横顔／全身シルエット／料理する姿／赤いワンピース・白エプロン・濃紺の髪／版画調・手描き風／3色程度／抽象化しすぎない／1色刷りでも成立／細線を避ける、等の条件で検証。狙いは「マスコットではなく**看板娘ロゴ**」。
5. **実用性の検証** — ヘッダー/フッターでの視認、1:1比率運用、シンボルを上＋店名2行＋タグライン、文字を斜めにしない、小サイズでも潰れない等。"絵"から"システム"へ。
6. **表情・人格の発見**（山場）— 笑顔ではなく、**目が合う、少し驚いた表情の看板娘**。「いらっしゃいませ！」ではなく「え、見つかっちゃった？」のような物語のある余白。可愛いけど媚びない／上品だけど冷たくない／レトロだけど古くない。このページで**演出的に一番見せ場**にする（フルスクリーン＋静かな余白）。
7. **髪型・服装・実写化の検討** — パーマヘア／赤いバンダナ／赤チェックシャツ／白エプロン／青スカートorジーンズ／三角巾バージョン（特に反応良）。さらにロゴの看板娘を**リアル写真風キービジュアル**へ（カフェ背景をぼかし人物にフォーカス）。「ロゴ→キャラクター→広告ビジュアル→ブランド世界」への展開。
8. **ビジュアルトーンの整理** — パリの空気感／クラシックすぎない現代ヨーロッパ感／手作り・洋食感／クリーム色・赤・濃紺の少数色／版画調・紙のかすれ／ベタ塗りすぎない／女性的でエレガント／日本的なのっぺりは避ける。**「記号で殴らない」**（エッフェル塔やベレー帽に逃げず、線・色・余白・佇まいでヨーロッパ感）。
9. **資料化とプレゼン** — ブランドの人格・世界観・ビジュアル方向性・展開可能性まで、短期間（約2日 ※要確認）で見える資料に。洋輔さんへ提案し**一発OK**で方向性確定。「この資料だけで100万の価値」と評された逸話　※要確認（誇張に見えないよう扱いは慎重に）。

## 飽きさせないためのレイアウト&演出（必須）

縦に長い分、**緩急で読ませる**。既存サイトのモーション文法の範囲内で：

- **ブロックの密度を交互に**：フルスクリーン画像 → 左右分割（テキスト＋画像）→ ステップ番号付きテキスト → 余白の引用（短い一文だけの"静"の章）→ 画像グリッド…を織り交ぜ、同じ型を3つ以上連続させない。
- **ステップ番号の演出**：各プロセスに `{01}〜{09}` のような連番（既存 mono ラベルの作法）。スクロールで番号が切り替わる/固定される等の軽い仕掛けで「今どの工程か」が分かるように（重い実装は GSAP ScrollTrigger、軽い reveal は Framer Motion）。
- **山場（ステップ6・驚き顔）** は、前後を広く空けてフルスクリーンで"間"を作り、ページ中盤のピークにする。
- 画像は**ken-burns / 軽いパララックス / reveal** の数種類に絞る（新種のド派手モーションは足さない）。
- スクロール進捗バー or 章インジケータがあると長尺で迷子になりにくい（既存にあれば踏襲、なければ控えめに1つ）。
- `prefers-reduced-motion: reduce` で全演出停止。画面外の重い処理は IntersectionObserver で停止（既存方針）。長尺＝画像多なので `next/image` の lazy + blur を徹底。

## 画像の扱い（重要：今回はすべてダミーで組む）

- **画像はすべてダミー（プレースホルダ）で配置し、レイアウトだけ完成させる。** ユーザーが後で差し込む。
- ダミーは `picsum.photos` の seeded URL 等でアスペクト比だけ正しく確保し、各所に **`// TODO: replace with real image →` のコメント＋想定内容**（例：`// TODO: ロゴ探索の生成バリエーション群`）を残す。
- **本番画像の配置先フォルダはこのパスを参照する**こと（コード内のコメント／`lib` のメモに明記）：
  ```
  /Users/temma/Library/CloudStorage/Dropbox/UTUTU/CorporateWebsite/public/images/brands/mademoiselle-croquette
  ```
  実運用では `public/images/brands/mademoiselle-croquette/` 配下になる想定。各プロセスstep用の画像が複数入る前提で、`mademoiselle-croquette/01-business/`, `/06-expression/` のように**ステップ単位のサブフォルダ命名**を提案し、データ側にその想定パスを仮置きする（ファイル名は後で確定）。
- **各プロセスに画像が複数枚入っても破綻しないギャラリー/グリッド**を用意（1枚ヒーロー、2枚並び、3〜N枚グリッド、を type で選べるように）。プロセスによって枚数が違う前提。
